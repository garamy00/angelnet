UpNote

UpNote 동기화는 macOS 의 upnote:// URL 로 본인 컴퓨터의 UpNote 앱을 호출해 새 노트를 만드는 방식입니다. 별도 계정/API 키 설정이 없으며, 앱에 로그인된 본인 계정의 로컬 노트북에 자동 저장됩니다. (외부 서버로 전송되지 않음)

체크 해제 시 메인 페이지의 🔄 UpNote 저장 (주) 버튼이 숨겨집니다.

UpNote 의 note/new 의 notebook 파라미터에 전달할 노트북 이름. 하위 노트북은 슬래시(/) 로 경로 표기 (예: 업무/2026). 비워두면 UpNote 기본 노트북에 저장됩니다.

체크하면 본문을 ``` 로 감싸 markdown 자동 변환 (들여쓰기 제거, * → bullet, 자동 번호 등) 을 모두 비활성화합니다. plain text 형식을 정확히 보존하려면 체크.

UpNote 의 markdown 파라미터. 일부 버전에서는 false 라도 자체 변환이 적용될 수 있으니, 결과가 어색하면 위 "코드 블록으로 감싸기" 도 체크하세요.

Notion

Notion 의 Database 1개에 엔트리(카테고리) 1개당 1행 으로 동기화합니다. UpNote 와 달리 동일 (날짜·카테고리) 행은 자동 업데이트되어 중복이 생기지 않습니다. UpNote 와 함께 사용해도 되고, Notion 만 사용해도 됩니다.

체크 해제 시 메인 페이지의 🔄 Notion 동기화 (주) 버튼이 숨겨집니다.

notion.so/my-integrations 에서 Internal Integration 생성 후 받은 token. 비밀번호처럼 macOS Keychain (service=angeldash-notion) 에 저장되며 화면에 다시 표시되지 않습니다. 토큰을 지우려면 빈 값으로 저장.

Notion Database 페이지 URL 끝의 UUID. Database 메뉴(···) → Connections 에서 위 Integration 을 연결해야 동기화가 가능합니다. DB 의 properties 는 아래 이름과 일치해야 합니다 (기본값 그대로 쓰는 걸 권장):

Property 이름 커스터마이즈

Projects (프로젝트 목록 push)

AngelDash 의 등록된 프로젝트 목록을 Notion 의 별도 DB 에 1프로젝트=1페이지로 push 합니다. 신규 항목만 추가되고 기존 페이지는 그대로 유지되므로, Notion 에서 wiki 처럼 작성한 내용이 보존됩니다. 프로젝트 페이지에서 🔄 Notion 업데이트 클릭으로 실행. Database 의 properties: Name(Title) — 그 외 WorkType(Select) 있으면 자동 채움.

Projects property 이름 커스터마이즈

Code 컬럼은 회사 타임시트의 task ID (remote_id) 가 들어갑니다. 기존 페이지의 Code 가 비어 있으면 자동으로 채워주고 (본문은 절대 안 건드림), 이미 값이 있으면 유지합니다.

Week Summary (주간 메모 동기화)

엔트리 동기화와 별개로, 메인 페이지의 📝 이번 주 메모 를 별도의 Notion Database 에 주별 1행으로 저장합니다. 같은 주(YYYY-Www) 행은 자동 업데이트되고, 메모가 비어 있으면 동기화를 건너뜁니다. Database 의 properties: Week(Title), Date(Date). 본문은 page content 에 코드블록으로 삽입.

Week property 이름 커스터마이즈

출력 템플릿

Jinja2 문법. {{ entry.category }}, {{ entry.body }}, {{ yy }}, {{ ww }}, {{ days }}, {{ week_notes }} 등.
