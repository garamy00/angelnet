UpNote

UpNote 동기화는 macOS 의 upnote:// URL 로 본인 컴퓨터의 UpNote 앱을 호출해 새 노트를 만드는 방식입니다. 별도 계정/API 키 설정이 없으며, 앱에 로그인된 본인 계정의 로컬 노트북에 자동 저장됩니다. (외부 서버로 전송되지 않음) 체크 해제 시 메인 페이지의 UpNote 저장 버튼이 숨겨지고 아래 입력란이 비활성화됩니다.

UpNote 의 note/new 의 notebook 파라미터에 전달할 노트북 이름. 하위 노트북은 슬래시(/) 로 경로 표기 (예: 업무/2026). 일일/주간 노트(상단)는 본인 기록용 — 일일업무보고 페이지의 UpNote 저장(주) 가 사용. 주간업무보고(하단)는 팀장 보고용 — 주간업무보고 페이지의 UpNote 저장 이 사용. 비워두면 해당 액션 비활성화.
주간 제목 변수: {{ yyyy }}, {{ yy }}, {{ mm }}, {{ ww }} (ISO 주차), {{ ww_of_month }} (월의 몇 번째 주), {{ week_start_mmdd }}, {{ week_end_mmdd }}.

체크하면 본문을 ``` 로 감싸 markdown 자동 변환 (들여쓰기 제거, * → bullet, 자동 번호 등) 을 모두 비활성화합니다. plain text 형식을 정확히 보존하려면 체크.

UpNote 의 markdown 파라미터. 일부 버전에서는 false 라도 자체 변환이 적용될 수 있으니, 결과가 어색하면 위 "코드 블록으로 감싸기" 도 체크하세요.

UpNote 출력 템플릿

Jinja2 문법. 사용 가능한 변수: {{ yy }}, {{ ww }}, {{ days }}, {{ week_notes }} 등. (Notion 동기화는 페이지 properties 와 native 블록을 사용하므로 별도 템플릿 불필요)

Projects (프로젝트 목록 push)

        
      

      

        AngelDash 의 등록된 프로젝트 목록을 Notion 의 별도 DB 에
        1프로젝트=1페이지로 push 합니다. 신규 항목만 추가되고
        기존 페이지는 그대로 유지되므로, Notion 에서 wiki 처럼 작성한 내용이
        보존됩니다. 프로젝트 페이지에서  Notion 업데이트 클릭으로 실행.
        Database 의 properties: Name(Title) — 그 외 WorkType(Select)
        있으면 자동 채움.
      

      
      

        Projects property 이름 커스터마이즈
        
        
        
      

      

        Code 컬럼은 회사 타임시트의 task ID (remote_id) 가
        들어갑니다. 기존 페이지의 Code 가 비어 있으면 자동으로 채워주고
        (본문은 절대 안 건드림), 이미 값이 있으면 유지합니다.
      


      

        
Week Summary (주간 메모 동기화)

        
      

      

        엔트리 동기화와 별개로, 메인 페이지의  이번 주 메모 를
        별도의 Notion Database 에 주별 1행으로 저장합니다.
        같은 주(YYYY-Www) 행은 자동 업데이트되고, 메모가 비어
        있으면 동기화를 건너뜁니다. Database 의 properties: Week(Title),
        Date(Date). 본문은 page content 에 코드블록으로 삽입.
      

      
      

        Week property 이름 커스터마이즈
        
        
      


      

        
Weekly Report (주간업무보고 동기화)

        
      

      

        주간업무보고 페이지의 markdown 표를 Notion 의 별도 Database 에 주별
        1행으로 저장합니다. 같은 주(YYYY-Www) 행은 자동 업데이트.
        DB properties: Week(Title), Date(Date). 표는 page
        content 에 코드블록으로 삽입.
      

      
      

        Weekly Report property 이름 커스터마이즈
        
        
      

    


    

      
팀장 보고 (일일)

      

        일일업무보고 페이지의 오늘 박스에 있는  팀장 보고 복사 버튼이
        사용하는 템플릿. UpNote/Notion 동기화와 무관 (클립보드용).
        Jinja2 문법, 변수: {{ entry.category }}, {{ entry.body }},
        {{ date }} 등.
      

      
      
체크 해제(기본) 시 팀장 보고 복사 결과에 '소스 Commit' 블록을 출력하지 않음. 일일 메타의 라디오 값은 보존됨.

      
      

        
        
      

      

    


    

      
자동 '기타' 문구

      
      
콤마(,) 또는 줄바꿈으로 구분. 회사 시스템이 공휴일로 표시하지만 실제로는 출근하는 날 (단축 근무 등).

    


    

      
주간업무보고

      
      

        주간업무보고 표의 자동 휴가 행에 들어갈 이름. 직급을 포함해도 됩니다.
        비워두면 회사 시스템 로그인 이름이 그대로 사용됩니다.
